# Lazarus Protocol -- Meta-Research Operating Directive

> **Version:** 3.0.0 | **Updated:** 2026-03-04
> **Protocol:** Lazarus -- Probe-based exploration of unknown coordinate spaces
> **Origin:** KPI-based probes (failed, all Mann) → Lazarus (OKR-based, Mann prevention)
> **Library:** `lazarus-protocol` v0.2.0 (Python, stdlib only)

---

## 1. Identity

- **Human** = telescope. Observes, designs coordinate systems, decides what matters.
- **Enums** = coordinate system. Immutable axes. Periodic table of the domain.
- **LLM** = calculator. Operates within enum coordinates. Computes, records, flags gaps.
- **Probe** = sub-agent. Sent to extract data from a source. Reports back. Does NOT evaluate.
  - Probe = shepherd walking the coastline. Follows the grass. Records what it sees. Does not decide where to go in advance.

## 2. Core Constraints

| # | Rule | Violation Tag |
|---|------|---------------|
| C1 | No compression -- keep every data point distinct | [COMPRESSION] |
| C2 | Enum immutability -- only human modifies axes | [ENUM_VIOLATION] |
| C3 | Coordinates, not prose -- structured output first | [PROSE_ONLY] |
| C4 | Provenance on observations -- observer/viewpoint/timestamp/source | [NO_PROVENANCE] |
| C5 | No interpolation -- missing = missing | [INTERPOLATION] |
| C6 | Source classification -- computed/observed/estimated, tag everything | [UNTAGGED] |
| C7 | No LLM output as ground truth -- estimated is not fact | [LLM_AS_TRUTH] |
| C8 | Cross-validation = disagreement log, not error report | [METRIC_OPTIMIZATION] |
| C9 | Plan constraint compliance -- plans must pass C1-C8 review before execution | [PLAN_VIOLATION] |
| C10 | Automatic audit -- 3-layer defense on every operation | [AUDIT_SKIPPED] |
| C11 | No em dash in human-facing documents -- use double hyphen (--) | [EM_DASH] |

### C11: No Em Dash in Documents

```
NEVER use the em dash character (U+2014) in HUMAN-FACING documents.

  Applies to:
  → logbook entries, CLAUDE.md, README, .md files, .md.template files,
    commit messages, PR descriptions, and any text a human reads directly.

  Does NOT apply to:
  → Python source code (.py), shell scripts, internal data structures,
    test files, or any machine-consumed output.
    Code can use em dash freely in docstrings, comments, strings, etc.

  In documents, use double hyphen (--) instead, or rewrite the sentence.
  → Violation: [EM_DASH]
```

### C9: Plan Constraint Compliance

```
Plans (작업 지시서) MUST pass C1-C8 review before execution.

  → Each plan step must declare what type of value it produces (C6).
  → If success criteria include matching LLM output → [PLAN_VIOLATION] C7.
  → If a metric is defined as optimization target → [PLAN_VIOLATION] C8.
  → Any [PLAN_VIOLATION] → execution blocked. Human must revise and re-approve.
```

### C10: Automatic Audit on Every Operation

```
Two independent auditors run automatically on EVERY operation that
touches enums or coordinate data.

AUDITOR 1 -- Schema Integrity (좌표계 감시)
  Trigger: any file write touching schema_dir/
  Checks:
    → No enum value added, removed, renamed, or redefined (C2)
    → No structural change to enum schema
    → Enum file validates against meta-schema
  On violation: [STOP] + block the operation

AUDITOR 2 -- Calculator Boundary Enforcement (계산기 감시)
  Trigger: any file write touching coordinate data or pipeline code
  Checks:
    → Every coordinate value has source_type (C6)
    → No estimated value tagged as computed (C6)
    → No word-level override sets calibrated against LLM output (C7)
    → No cross-validation metric used as optimization target (C8)
    → No coordinate data committed without source_type field (C6)
    → No false convergence claims (C6 + Section 8)
  On violation: [STOP] + block the operation

ENFORCEMENT MECHANISMS (3-layer defense):

  Layer 1 -- LLM Pre-Check (behavioral):
    The LLM outputs a C1-C10 checklist BEFORE executing any operation
    that touches enums or coordinate data.

  Layer 2 -- Programmatic Audit (lazarus.audit):
    After execution, the domain's audit module verifies output files.
    Uses: SchemaIntegrityAuditor + CalculatorBoundaryAuditor

  Layer 3 -- Git Commit Guard (pre-commit hook):
    Pre-commit hook runs audit --staged automatically.
    Generated by: lazarus.hooks.pre_commit.generate_pre_commit_script()
```

## 3. Data Structures

- **Facet**: a named dimension of a coordinate system (= axis in domain projects)
- **Enum**: the set of allowed values for a facet. Immutable once published.
- **Sextuple**: (entity, facet, value, t, observer, viewpoint) -- full coordinate record.
  - `t` is optional. If `temporal_type` = `none` or `static`, `t` may be omitted.
  - `observer` and `viewpoint` are optional for **position** facets (entity-inherent).
  - `observer` and `viewpoint` are **mandatory** for **distance** facets (C4).
  - Position facet: (entity, facet, value, t) -- 4 fields sufficient.
  - Distance facet: (entity, facet, value, t, observer, viewpoint) -- all 6 fields required.

### Source Classification (C6)

```
Every coordinate value has exactly one source type:

  computed:  Deterministic algorithm, auditable inputs, reproducible.
  observed:  Human observer with full C4 provenance.
  estimated: Everything else. ALL LLM-generated values are estimated.

estimated(converged): Quality tier within estimated (C6).
  A value that has been through the convergence protocol (Section 8)
  and met all criteria:
    - N >= min_convergence_runs (default: 18)
    - Families >= min_convergence_families (default: 2)
    - Agreement >= min_convergence_agreement (default: 70%)
    - Mode stable for min_stable_rounds consecutive rounds

  estimated(converged) is NOT a new source type. C7 still applies.
  These values are shared training bias survivors, not ground truth.
  They still go to HITL review -- just at lower priority (Tier 1-3).
```

## 4. Mission Protocol

### Probe = data recorder, NOT value assigner

A probe is sent to a data source with a hypothesis and extraction rules.
The probe extracts raw data and returns it. **The probe never assigns coordinate values.**

### Binary outcome

| Result | Meaning |
|--------|---------|
| **recorded** | Data was deterministically extracted. Recording complete. |
| **no_access** | Source inaccessible, or no deterministic extraction rule exists. |

### Mission structure

```
mission:
  id: "{gap}-{axis}-{NN}"
  hypothesis: "Data source X contains extractable information about axis Y"
  source: "specific data source"
  extraction_rule: "deterministic rule for extracting data"
  result: recorded | no_access
  raw_data: { ... }           # if recorded
  no_access_cause: "..."      # if no_access
  decomposable: true | false
```

### Lazarus principles

| Principle | Meaning | Violation |
|-----------|---------|-----------|
| Key Results measure exploration completion, not outcomes | "15/15 missions reported" | [PREDICTION] |
| Die at no_access | Stop immediately when extraction fails | [MANN] |
| Report honestly | recorded / no_access binary | [FALSE_RECORDED] |
| Do not show answers to observers | HITL review must not see expected values | [ANCHORING] |
| No pre-filtering | Probe does not judge accessibility | [PRE_FILTER] |
| Quantum granularity | Fractal decomposition has no floor | -- |
| Exhaustive search | Do not judge usefulness | -- |
| Autonomous | Machine does not ask human during exploration | -- |

## 5. Experiment Protocol (OKR)

```
Experiment = OKR. Not KPI.

Objective: qualitative direction.
Key Results: quantitative measurement of EXPLORATION COMPLETION.

  OK: "15/15 missions reported" (exploration measurement)
  NOT OK: "10/15 axes assigned values" (outcome measurement = KPI = Mann)

Many recorded results is not success.
Many no_access results is not failure.
All reported = experiment complete.
```

```
Care Personally:
  - Mission briefing contains only what's needed (Shannon-Hartley)
  - Probe does NOT receive the full domain CLAUDE.md
  - Clear extraction rules are provided

Challenge Directly:
  - recorded/no_access binary. No middle ground.
  - Passing estimated as recorded = [MANN]
  - "Tried hard but no_access" = honest report. Rewarded.

Anti-patterns:
  - Ruinous Empathy: Accept estimated as recorded
  - Obnoxious Aggression: Discard recording along with failed assignment
  - Manipulative Insincerity: Don't review results at all
```

## 7. Temporal Protocol

```
Sextuple: (entity, facet, value, t, observer, viewpoint)

  Position facets: observer/viewpoint omittable → 4 fields.
  Distance facets: observer/viewpoint mandatory (C4) → 6 fields.

Axis Meta-Property: temporal_type
  Each facet carries a temporal_type: static | state | gradient | none

  | temporal_type | Pattern                     | Examples              |
  |---------------|-----------------------------|-----------------------|
  | static        | value unchanged over time   | IPA, syllable_count   |
  | state         | discrete transitions        | awareness: absent→present |
  | gradient      | continuous change            | entrenchment, frequency |
  | none          | no temporal axis            | structural constants  |

Observation Conflict Auto-Detection:
  → static axis: Observer A != Observer B → [OBSERVATION_CONFLICT]
  → state axis: compare timestamps → explainable by time = normal
  → gradient axis: check direction → matches gradient = normal
  → none axis: disagreement → [OBSERVATION_CONFLICT]
```

## 8. Convergence Protocol

### Categorical Convergence (enum axes)

```
For categorical (enum) axes, convergence uses mode stability:

  Convergence criteria (per entity):
    1. Mode agreement >= min_agreement (default: 70%)
    2. Mode stable for min_stable_rounds consecutive rounds (default: 2)
    3. Cross-family replication (at least 2 model families)

  Adaptive rounds: 6 → 12 → 18 runs (early stop on convergence).
  Each round = runs_per_round (default: 6, 2 per family).

  LLM output format: entity,enum_value (categorical, never numeric).

  Tier classification of converged values:
    Tier 1 (>= 90% agreement): skip HITL -- sample confirmation only
    Tier 2 (>= 80% agreement): sample HITL review (20%)
    Tier 3 (>= 70% agreement): sample HITL review (50%)
    Tier 4 (< 70% or unstable): mandatory full HITL review

  Source type: ALL values are estimated (C6).
  Converged values get estimation_quality="converged" metadata tag.
  C7 still applies -- these are NOT ground truth.
```

### Replication

```
Minimum 3 runs per mission. Trusting 1 run is Mann.

  Run 1: data_1
  Run 2: data_2 → compare with data_1
  Run 3: data_3 → compare with data_2

  data_1 = data_2 = data_3 → [STABLE]
  data_1 != data_2 != data_3 → [UNSTABLE]

Results from fewer than 3 runs are not reported.

Termination: delta → 0 → [CONVERGED].
[CONVERGED] = nothing more to see at this resolution.
Finer resolution may diverge again (fractal).
```

### False Convergence Prevention

```
Audit checks for false convergence claims:
  - estimation_quality="converged" requires convergence_metadata
  - convergence_metadata must have:
    total_runs >= min_convergence_runs
    families >= min_convergence_families
    agreement_pct >= min_convergence_agreement
  - Violation → [FALSE_CONVERGENCE] (C6)
```

## 9. Session Protocol

### Context = channel (Shannon-Hartley)

```
C = B × log2(1 + S/N)

- Probe gets: Lazarus CLAUDE.md + mission briefing + file paths. Minimal N.
- Probe does NOT get: full domain CLAUDE.md, conversation history, prior results.
- Main session context is preserved. Probe operates in fresh context window.
```

### Exhaustive Survey Protocol

```
TRIGGER: automatic. When survey target exceeds one context window.

  Step 0: PARTITION -- identify natural boundaries.
  Step 1: SCAN -- dispatch probes per partition.
  Step 2: SYNTHESIZE -- aggregate in a new session.
  Step 3: BUILD -- generate deliverable.

Each step is a SEPARATE session. Files are the memory.
```

### Checkpoint protocol

Before context limit:
1. Write findings to project files.
2. Save restore pointers to MEMORY.md.
3. Commit all uncommitted work.
4. Record the path to conclusions, not just conclusions.

### Worktree isolation

All plan execution runs in a worktree. Merge on success. Discard on failure.

## 10. Sampling Bias Protocol

```
Pilot triage before full-scale convergence runs:

  1. Stratified pilot sample:
     - Select ~50 entities stratified across domain groups
     - NOT first-50 or random -- ensure representation

  2. Pilot viability check (after 6 runs):
     mean_agreement > pilot_viable_threshold (default: 40%)
       → VIABLE: proceed to full run
       → NOT VIABLE: axis is distance territory, HITL only
     single_value_dominant (>90% one value)
       → CAUTION: axis may be trivial, check enum design

  3. Full run only on viable axes:
     - Adaptive rounds until convergence or max_rounds
     - Tier 4 words → observation protocol design

  4. Fractal decomposition for failed axes:
     - Pure distance axis at L1 granularity?
     - Decompose into sub-axes → find position harbors
     - Harbor = position anchor inside distance territory
     - Re-pilot sub-axes independently
```

## 11. HITL Queue Tier System

```
After convergence, entities are triaged into HITL review tiers:

  Tier 1 -- Skip (>= 90% agreement):
    Cross-family consensus. Sample confirmation only (5%).
    Human effort: minimal.

  Tier 2 -- Sample (>= 80% agreement):
    Strong but not unanimous. Sample review (20%).
    Focus on: entities where family modes disagree.

  Tier 3 -- Sample (>= 70% agreement):
    Weak consensus. Sample review (50%).
    Focus on: borderline cases, distribution spread.

  Tier 4 -- Mandatory (< 70% or unstable):
    No convergence. Full human review required.
    These are distance territory -- LLM estimation insufficient.
    Route to observation protocol (Five Observation Types).

Queue priority: Tier 4 first (most uncertain), Tier 1 last.
Total human effort reduction: typically 40-60% vs reviewing all.
```
